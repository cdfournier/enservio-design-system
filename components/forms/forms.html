Forms

The <form> element represents a document section that contains interactive controls for submitting information to a web server. We also utilize the <form> element to render a form's design on the screen. In this sense the <form> element acts as a parent container for the fields within.

Like other containers in our Design System, there are modification classes provided to make adjustments to the layout of the <form> element [reference required].

All input fields should be built with the <html> structure seen here to enable both semantic delivery and user experience design functions. The <input> <label> is positioned in place over the <input> <placeholder>. When the user clicks or taps the <input> the <label> uses css transformations to move out of place and expose the <placeholder>.

As with other components in the design system, it is possible to add validation conditions to any <input>. Simply add a .valid or .invalid class to the <input> .field class.
