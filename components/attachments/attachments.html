Attachments

There are screens for Claim Attachments and for individual Inventory Item Attachments. The two screens look and function alike, the only difference being that the screen for individual Inventory Item Attachments includes the inventory item header from the Inventory Capture screen (included below).

The are two functional components on these screens: the upload button and container and the attachments list.

Drag Files Here To Upload

PDF, PNG, JPG, DOC

We utilize Modernizr to detect touch-screen devices which will set the <div class="upload-container"> to display: none;. When no touch-screen is detected, we apply the .no-touch class to the upload container to make it visible for drag-and-drop functionality.

On mobile devices, the Add Attachments button should enable the device's native file features, including access to the device's camera, photo album, and file management system.

The attachment list is an unordered list whose individual line-items with two additional children components including a <div> and a <span> (for error messages).

The <div class="line-item"> is split into three child elements: a <span> for the .file-type-icon, a <span> for the .file-name, and a Button Box for the delete line item button.

To display error messages for each <li>, add the class .invalid to the <li>.

All file-type icons can be found here: Line Awesome.
