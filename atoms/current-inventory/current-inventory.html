Current Inventory

Items added to the user's claim inventory (done by clicking the Add Item To Attachment button in the new item Inventory Capture form) are stored in the Current Inventory.

On small screens (with less than 768 pixels of vertical height), the Current Inventory section is pinned to the bottom of the screen. When users click the Current Inventory Header, the section should expand up and fill the entire screen.

See the Inventory Capture demo page to examine this behavior.

Only one current inventory item may be open at a time. If a user makes any changes to a current inventory item, they must save or cancel those changes before being able to open another item to edit.

Submitted Current Inventory List Item Header

If a user has previously clicked the Submit Inventory button and has returned to enter additional inventory items, those items previously submitted should still be shown in their Current Inventory list. They should, however, be disabled, and the user should be prevented from making any changes to these items. The user may also not delete these items from their inventory. They may click the Attachments button to view the list of attachments for this inventory item, but they should not be able to add new attachments or delete those items already attached.
